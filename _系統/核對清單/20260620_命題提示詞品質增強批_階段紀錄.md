## 七、A/B 驗收建議

任務四正式 A/B 時，建議區分兩種新版：

| 新版類型 | 說明 | 適合用途 |
|---|---|---|
| B1：規則增強版 | 不放 few-shot，只測誘答規格與 `qualityMeta` | 測 prompt 規則本身是否有效 |
| B2：few-shot 增強版 | 放入已准入的 few-shot 範例 | 測完整品質增強效果 |

建議流程：

1. 先跑 A vs B1：看純規則與 `qualityMeta` 是否讓題目品質提升。
2. 再跑 B1 vs B2：看 few-shot 範例是否額外提升題感。

如此可避免把 prompt 規則效果與 few-shot 題感效果混在一起。


但 Run 1 唯一 hard fail 是 B 組 `G4_AB_MA_001 / R011`，理由為與 A 組同 slot 題目內容重複。後續只讀復核確認：

- 題幹清楚、正答唯一、符合 slot / `primaryObjectiveId`。
- 誘答具診斷性，學生版解析基本合理。
- B 組該題 `qualityMeta` 全綠。
- hard fail 主要來自 A/B 跨版本同 slot 重複造成的評分 artifact。

因此 Run 1 不作為最終導入判定；其結論改為「暫定 C」。工程決策維持不變：PR #1 繼續 Draft，不 merge、不 deploy。

本次文件修正調整 A/B 盲評規則：

1. 新增 `duplicateObservation` 欄位。
2. 明確區分「跨版本同 slot 相似觀察」與「真正 hard fail」。
3. 規定 `duplicateObservation: cross_version_same_slot` 不得直接判 hard fail。
4. 只有同一份測驗內重複、近乎複製且缺乏獨立評量價值、明顯抄襲 few-shot/公開題、或重複造成學生作答/評量失真時，才可因重複判 hard fail。
