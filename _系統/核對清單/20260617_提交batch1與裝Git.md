# 核對清單：裝 Git → 驗證 → 提交 batch 1（孤兒題／timeout 修補）

- 日期：2026/6/17
- 目的：把現有未提交的修補封成一個 commit，讓工作樹乾淨，再開 Phase 1
- 全程在你本機（Windows）執行；commit／push 由你跑

---

## 步驟 0：安裝 Git（擇一）

- [ ] **Git for Windows**：https://git-scm.com/download/win （一路 Next 預設即可）
- [ ] 或 **GitHub Desktop**：https://desktop.github.com/ （內含 git，且有圖形介面）

裝完開新的 PowerShell 或 VS Code 終端機，確認：
```powershell
git --version
```

---

## 步驟 1：進到專案、清掉殘留鎖檔（保險）

```powershell
cd D:\User\Nhps\Documents\exam-wizard-next
if (Test-Path ".git\index.lock") { Remove-Item ".git\index.lock" }
```

---

## 步驟 2：首次用 git 需設身分（若已設過可略）

```powershell
git config user.name "smallcannon-arch"
git config user.email "你的 GitHub 帳號信箱"
```

---

## 步驟 3：驗證（四道全過才提交）

```powershell
npm test
npm run check
node --check worker/src/prompts.js
node --check worker/src/gemini.js
```

- [ ] `npm test` 全綠
- [ ] `npm run check` 通過
- [ ] 兩個 `node --check` 無輸出（即無語法錯）

任一不過先別提交，把訊息貼給我。

---

## 步驟 4：移除被追蹤的 9 個 desktop.ini（檔案留在硬碟，只是不再追蹤）

```powershell
git rm --cached desktop.ini docs/desktop.ini frontend/desktop.ini frontend/src/desktop.ini frontend/src/core/desktop.ini scripts/desktop.ini tests/desktop.ini worker/desktop.ini worker/src/desktop.ini
```

---

## 步驟 5：只 stage 這一批（8 個程式碼檔 + 2 個設定檔）

```powershell
git add frontend/index.html frontend/src/apiClient.js frontend/src/app.js frontend/src/core/renderPaper.js frontend/src/core/validateGeneratedPaper.js tests/validateGeneratedPaper.test.js worker/src/gemini.js worker/src/prompts.js .gitattributes .gitignore
```

⚠️ **不要 stage 這三個**（只有 CRLF 換行差異、零內容變動）：
`frontend/src/config.js`、`frontend/src/core/normalizeItem.js`、`tests/normalizeItem.test.js`

---

## 步驟 6：確認 staged 內容後提交

```powershell
git status
```
- [ ] 確認 staged 只有步驟 4、5 那些（外加 desktop.ini 顯示為 deleted）
- [ ] 確認上面三個 CRLF-only 檔**不在** staged 區

```powershell
git commit -m "fix: 補閱讀測驗孤兒題檢核與作答括號、生成逾時緩解；設定 .gitattributes 統一換行碼"
```

---

## 步驟 7：推送（可選，你決定）

```powershell

```
