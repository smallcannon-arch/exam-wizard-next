## 〇、核心規格（總綱，一句話）

> 命題系統不應以**題型**為中心，而應以**題位（slot）**為中心；每個題位同時承載分數、題型、**主要能力歸屬**與生成約束。國語不再使用假節數，改由**向度比例**產生能力預算；題型只是呈現形式，**不能反推**能力歸屬。

---

## 一、問題的根：一份卷有兩條軸，現在被擠成一條

- **能力／內容軸**（縱軸，支撐內容效度）：要考什麼能力／指標。國語＝向度／細項；其他科＝學習目標。
- **形式軸**（橫軸，影響評量形式均衡與作答／評分穩定性）：用什麼題型呈現。

這正是評量學的「雙向細目表」。兩軸本來獨立，現行系統卻用題型自動推導向度（`getChineseDimension`），等於「以形式限縮內容」，損害效度。國語的「假節數（`｜1`）」與閱讀權重失真，都是這個錯置的下游症狀。

---

## 二、普世原則：全科同一管線，抽換零件（非國語特例）

```
選能力項 → 給權重 → 算配分預算 → 題位設計（含題型）→ AI 逐題位生成
```

**反 `if (subject==='國語')` 寫法，改用 Subject Profile：**

```js
const profile = getSubjectProfile(subject);
```

| 科目 | 能力框架 abilityFramework | 權重策略 weightStrategy | 配題策略 blueprintStrategy |
|------|--------------------------|------------------------|---------------------------|
| 國語 | chineseDimensionItems（國語評量向度細項模板） | dimensionRatio（向度比例） | reverseFromAbility（反向，後期） |
| 自然 | learningObjectives | lessonHours（節數） | manualItemTypes |
| 數學 | learningObjectives | lessonHours | manualItemTypes |
| 社會 | learningObjectives | lessonHours | manualItemTypes |
| 英文 | skills（聽說讀寫字彙） | skillRatio 或 lessonHours | manualItemTypes（先不反向） |

來源參照（僅開發備忘，不寫進程式／UI）：國語細項模板參照許育健國語試題檢核表／校內審題規準；程式命名採中性 `chineseDimensionItems`，不得用人名命名（如 `xuYuJianItems`）。

每個 profile 定義：能力框架、權重怎麼算、是否反向生成題型、題位如何對應能力、偏離怎麼示警。

## 三、三層模型（把原本的兩層講清楚）

1. **能力預算層**：這份卷考什麼？各占幾分？（國語＝向度比例；他科＝目標節數比例）
2. **題位設計層（核心）**：有哪些題位？每題幾分？每題掛哪個能力項？題型是什麼？
3. **AI 生成層**：依題位規格逐題生成，不自由發揮整卷。

題位範例（單一真相來源的最小欄位）：

```json
{
  "slotId": "S001",
  "score": 2,
  "itemType": "multiple_choice",
  "primaryAbility": {
    "dimension": "段篇讀寫",
    "abilityId": "提取訊息",
    "label": "提取訊息"
  },
  "abilityRefs": ["提取訊息"],
  "groupId": "G001"
}
```

| 欄位 | 用途 |
|------|------|
| `primaryAbility` | 統計用（總分、向度比例皆據此），第一階段核心 |
| `abilityRefs` | 保留擴充彈性，可先等同 primary |
| `secondaryAbilities` | 跨向度審題用，**第二階段再做** |

**abilityId 穩定鍵護欄（封盤拍板）**：Phase 1 暫以穩定中文細項名稱作為 `abilityId`，避免過早建立中英代碼表；`abilityId` 一經進入題位資料，即視為**穩定資料鍵**，不得因 UI 文案潤飾任意變更（要潤飾改 `label`）。未來若公開化或跨系統整合，再升級為代碼化 ID，並提供遷移對照表（migration map），否則舊題位統計會斷。

---

## 四、題組：本身不配分，只有子題配分

題組＝共用素材 + 數個子題；配分與向度**歸屬到子題**，不歸屬到大題。

```json
{
  "groupId": "G001",
  "groupType": "reading_set",
  "sharedMaterial": { "type": "text", "title": "短文閱讀" },
  "slots": [
    { "slotId": "S001", "score": 2, "itemType": "multiple_choice", "primaryAbility": { "dimension": "段篇讀寫", "abilityId": "提取訊息", "label": "提取訊息" } },
    { "slotId": "S002", "score": 2, "itemType": "multiple_choice", "primaryAbility": { "dimension": "段篇讀寫", "abilityId": "推論訊息", "label": "推論訊息" } },
    { "slotId": "S003", "score": 2, "itemType": "multiple_choice", "primaryAbility": { "dimension": "句式語法", "abilityId": "文句辨析", "label": "文句辨析" } }
  ]
}
```

這直接解掉「閱讀測驗一大題算幾分、算哪個向度」與「72 ≠ 100」的病根。

## 五、學力檢測題跨向度：主歸屬 + 次標記（次標記先緩做）

- 每個子題只掛**一個 primary 向度／細項**（統計用此）。
- 可選 `secondaryAbilities`（審題提醒用，**第一階段先不做**，避免 UI 過早複雜）。
- **不要**做「2 分拆成 0.8/0.7/0.5」的精細加權，老師看不懂、UI 難維護。

---

## 六、向度偏離示警：分級提醒，不硬擋

| 偏離程度 | 系統反應 |
|---------|---------|
| ±5% 內 | 正常 |
| ±5%~10% | 黃色提醒 |
| 超過 ±10% | 紅色警示，但允許教師確認後繼續 |

文案示例：「段篇讀寫目前 12%，低於建議 20%。建議增加閱讀理解、段落推論或短文表達題位。」

制度定位：學習評量由授課教師依課程目標、學生差異與多元評量原則辦理；新竹市學校並設學生學習評量輔導小組及校內補充規定，作為評量與學習輔導事項之研議、審查機制。系統定位為「專業導航」，提供推薦值與偏離提醒，但不取代教師命題判斷與校內評量規範。
（法規原文與條號，封盤對外前由 NotebookLM 核對母法《國民小學及國民中學學生學習評量辦法》與新竹市補充規定。）

## 七、落地順序（修正版：先扶正模型，再談聰明）

> 鐵則（時序）：**現有那批未提交的孤兒題／timeout 修補，先驗證、提交、封盤（乾淨樹），才開始本重構。** 不可把脊椎重構疊在暫存修法上。

**Phase 1：題位＝單一真相來源（全科共通地基）**
- 每題位含 `slotId / score / itemType / primaryAbility（dimension+abilityId+label）/ abilityRefs`。
- 題組只含 `sharedMaterial` + 子題；大題不自己配分。
- **所有統計（總分、向度比例、題型分布、目標覆蓋率）一律從題位重算**。
- 修掉「題型推向度」與「假節數 `｜1`」。
- 順手修「配題分布表 72 ≠ 100」：顯示層也改從題位重算（此項可先行）。
- 黃金準則：`Σ(所有子題配分) ＝ 試卷總分`。
- 遵守 abilityId 穩定鍵護欄（見第三節）。

**Phase 2：Subject Profile 抽象化**
- 建 `subjectProfiles/`，把權重策略與能力框架做成可插拔設定。國語從此不必假裝有節數。

**Phase 3：國語向度比例預算**
- 依年級建議比例（低 50/30/20、中 30/50/20、高 20/30/50）或教師自訂，算出各向度應佔分；題位統計目前落點。

**Phase 4：向度偏離示警**
- 從題位重算落點，套第六節分級燈號。

**Phase 5：國語反向配題表（細項→題型對照表才在此登場）**
- 勾細項 → 建議題型 → 產生題位初稿（**完全可編輯**）。
- 對照表「向度細項模板引導 + 教師自訂覆寫」：字音→注音、字形→國字、提取訊息→閱讀測驗等緊耦合對應內建為預設，老師可覆寫。
- 可覆寫原則：老師改題型，只動題位的「形式標籤」，保留其「能力歸屬」；除非連能力一起改，向度比例不崩。

> 註：你的程式碼**已有半套題位模型**（slot 已帶 `primaryObjectiveId`、`chineseDimension`、`isGroup`、`subScores`；`validateGeneratedPaper` 已按 parent 分組加總子題配分）。本重構是「扶正既有題位成唯一真相、拔掉題型推向度與假節數」，是演化不是重寫。

## 八、五個自問的拍板防線

1. **向度比例可否逐卷微調？** 可，但提供推薦值；偏離過大（如高年級段篇 <10%）跳專業提醒。制度定位：評量由授課教師辦理，新竹市另設學生學習評量輔導小組與校內補充規定為審查機制。
2. **學力跨向度歸屬？** 切到子題層級，每子題一個 primary 向度；secondary 供審題、先緩做。
3. **細項→預設題型誰定？** 向度細項模板內建預設 + 教師覆寫。
4. **英文要反向驅動嗎？** 先不要，維持鬆耦合手動布題；profile 預留 `manualItemTypes`，國語穩了再評估半反向。
5. **鬆耦合科要顯示能力涵蓋率嗎？** 要。雖按節數鋪題位，仍可提示「已涵蓋本單元 80% 指標，『昆蟲的構造』尚未出題」，當命題安全網。
